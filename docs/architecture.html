범용 국회 모니터링 자동화 시스템의 전체 데이터 파이프라인과 워크플로우

1 전체 시스템 개요

유튜브 국회방송 영상을 자동 감지하고, 자막을 추출해 AI로 분석한 뒤, 구조화된 데이터를 대시보드에 표시합니다.

핵심 특징

  • 도메인 무관 — config.yaml만 바꾸면 어떤 산업이든 모니터링
  • AI 2-pass 분석 — 구조화(Pass 1) + 누락 검증(Pass 2)
  • 3단계 자막 Fallback — 안정성 보장

기술 스택

  • LLM: Claude (Anthropic API)
  • 저장: Google Sheets API (6개 탭)
  • 자동화: GitHub Actions (cron)
  • 대시보드: Vanilla JS + GitHub Pages

3 영상 감지 프로세스

video_detector.py — YouTube API와 수동 큐에서 영상을 수집하고 중복을 제거합니다.

flowchart TD subgraph AUTO["자동 감지 (YouTube API)"] direction TB COMMS["config.yaml\ncommittees 목록"] --> SEARCH["YouTube Data API v3\n검색어: {search_term} 국정감사"] SEARCH --> RESULTS["검색 결과\n(위원회별 최대 10개)"] end subgraph MANUAL["수동 큐 (대시보드)"] direction TB QUEUE["_manual_queue 탭\nstatus = pending"] --> PARSE["URL → video_id 추출"] end RESULTS --> MERGE["영상 목록 통합"] PARSE --> MERGE MERGE --> DEDUP{"_processed_videos\n탭에 존재?"} DEDUP -- "이미 처리됨" --> SKIP2["스킵"] DEDUP -- "신규" --> TASK["VideoTask 생성\nvideo_id, committee, date, source"] style AUTO fill:#eff6ff,stroke:#2563eb,color:#1e3a5f style MANUAL fill:#fefce8,stroke:#f59e0b,color:#713f12 style TASK fill:#dcfce7,stroke:#16a34a style SKIP2 fill:#fef2f2,stroke:#dc2626

VideoTask 구조

  • video_id — YouTube 영상 ID
  • committee / committee_code
  • date — 영상 날짜 (YYYY-MM-DD)
  • source — "auto" 또는 "manual"
  • event_type — 국정감사, 업무보고 등

중복 방지

이미 _processed_videos 탭에 기록된 video_id는 건너뜁니다. 수동 큐 영상은 상태가 pending → processing → done/error로 변경됩니다.

4 자막 추출 Fallback 체인

subtitle_extractor.py — 3가지 방법을 순차적으로 시도하여 자막을 확보합니다.

1차: youtube-transcript-api

  • 가장 빠르고 안정적인 방법
  • 한국어 자막 우선 (속기 > 자동생성)
  • 봇 차단에 비교적 강함

2차: NotebookLM

  • Google 인프라를 경유하여 IP 차단 우회
  • 비공식 API — 인증 필요
  • 180초 타임아웃, 100자 미만이면 실패 처리

3차: yt-dlp

  • CLI 도구 (subprocess 실행)
  • VTT 포맷 다운로드 → 파싱
  • 자동생성 자막의 중복/오버랩 제거

브라우저 전달 자막

  • 대시보드에서 직접 추출한 자막
  • gzip + base64 압축 해제
  • Fallback 체인을 건너뜀

5 AI 2-Pass 분석

text_processor.py + llm/prompts.py — Claude AI가 자막을 구조화하고, 누락된 도메인 안건을 검증합니다.

flowchart TD INPUT["자막 텍스트"] --> KWF{"키워드 필터\ninclude/exclude"} KWF --> CHUNK["텍스트 분할\n80KB 청크, 5KB 오버랩"] CHUNK --> P1["Pass 1: 구조화\n(청크별 실행)"] P1 --> |"system: 도메인 정의 + 키워드 + 기관명\nuser: 자막 텍스트"| CLAUDE1["Claude Haiku\n(retry: 3회, 5s 간격)"] CLAUDE1 --> R1["안건 목록\n각 안건: domain / general 분류"] KWF --> |"도메인 키워드 있음"| P2GATE{"general 안건 중\n도메인 관련\n누락 가능성?"} R1 --> P2GATE P2GATE -- "있음" --> P2["Pass 2: 검증\n(전체 1회)"] P2 --> |"system: 도메인 + 키워드\nuser: Pass1 결과 + 자막"| CLAUDE2["Claude Haiku\n(retry: 2회, 5s 간격)"] CLAUDE2 --> MERGE2["재분류 병합\ngeneral → domain 승격"] MERGE2 --> FINAL2["최종 안건 목록"] P2GATE -- "없음 / 키워드 없음" --> FINAL2 FINAL2 --> OUTPUT["구조화된 출력"] OUTPUT --> AG2["안건별: title, category,\nsummary, is_entity_mentioned"] OUTPUT --> ST2["발언별: speaker_name,\nparty, role, content"] style P1 fill:#dbeafe,stroke:#2563eb style P2 fill:#fef3c7,stroke:#f59e0b style CLAUDE1 fill:#f0fdf4,stroke:#22c55e style CLAUDE2 fill:#f0fdf4,stroke:#22c55e style FINAL2 fill:#dcfce7,stroke:#16a34a style OUTPUT fill:#ede9fe,stroke:#8b5cf6

Pass 1: 구조화

  • 자막에서 안건 경계를 식별
  • 각 안건을 domain / general로 분류
  • 발언자 정보(이름, 정당, 역할) 추출
  • 주요 기관/기업 언급 여부 기록

Pass 2: 검증

  • 도메인 키워드가 자막에 있을 때만 실행
  • general로 분류된 안건 재검토
  • 누락된 도메인 안건을 domain으로 승격
  • 실패 시 Pass 1 결과를 그대로 사용

청킹 전략

  • 80KB 이하: 단일 청크 처리
  • 80KB 초과: 5KB 오버랩으로 분할
  • 각 청크를 독립적으로 Pass 1 실행
  • 결과를 하나로 합쳐서 Pass 2

오류 복구

  • Pass 1: 최대 3회 재시도 (5s → 10s → 20s)
  • Pass 2: 최대 2회 재시도 (5s → 10s)
  • JSON 파싱 실패 시 자동 복구 시도
  • Pass 2 전체 실패 시 Pass 1 결과 유지

6 뉴스 검색

news_searcher.py — Naver Search API로 각 도메인 안건의 관련 뉴스를 검색합니다. (선택 기능)

flowchart LR AGENDAS["domain 카테고리\n안건 목록"] --> EACH["안건별 검색어 생성\n안건 제목 기반"] EACH --> NAVER["Naver Search API\n뉴스 검색"] NAVER --> FILTER["결과 필터링\n중복 제거, 관련성 확인"] FILTER --> ATTACH["안건에 뉴스 첨부\n(최대 5개/안건)"] ATTACH --> SHEETS2["news_articles 탭\n저장"] style AGENDAS fill:#dbeafe,stroke:#2563eb style NAVER fill:#f0fdf4,stroke:#22c55e style SHEETS2 fill:#ede9fe,stroke:#8b5cf6

동작 조건

NAVER_CLIENT_ID + NAVER_CLIENT_SECRET 환경변수가 설정된 경우에만 실행됩니다. 미설정 시 뉴스 검색 없이 파이프라인이 계속됩니다.

Rate Limiting

Naver API 요청 간 100ms 딜레이를 두어 rate limit을 방지합니다. 안건당 최대 5개 뉴스 기사를 수집합니다.
